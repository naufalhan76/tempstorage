API Documentation

Mabox provides a simple REST API for temporary file storage with automatic expiration

🔗 Base Information

Essential details for using the Mabox API

Base URL

http://localhost:3000
Feature Details
File Expiry 24 hours from upload
Max File Size 100 MB per file
Authentication None required
Rate Limiting Reasonable use policy

📤 Upload Files

Upload one or multiple files to temporary storage

POST /upload

Request

Send files as multipart/form-data:

curl -X POST http://localhost:3000/upload \ -F "files=@document.pdf" \ -F "files=@image.jpg"

Response

{ "success": true, "files": [ { "id": "abc123", "filename": "document.pdf", "originalname": "document.pdf", "size": 1024000, "uploadDate": "2025-09-18T10:30:00.000Z", "expiryDate": "2025-09-19T10:30:00.000Z", "downloadUrl": "/files/abc123/document.pdf" } ] }

📥 Download Files

Download files using their unique ID

GET /files/:id/:filename

Request

curl -X GET http://localhost:3000/files/abc123/document.pdf \ -o downloaded_document.pdf

Response

Returns the file content with appropriate headers:

  • Content-Type: Detected MIME type
  • Content-Disposition: attachment; filename="document.pdf"
  • Content-Length: File size in bytes

ℹ️ Get File Information

Retrieve metadata about a specific file

GET /files/:id/info

Request

curl -X GET http://localhost:3000/files/abc123/info

Response

{ "success": true, "file": { "id": "abc123", "filename": "document.pdf", "originalname": "document.pdf", "size": 1024000, "uploadDate": "2025-09-18T10:30:00.000Z", "expiryDate": "2025-09-19T10:30:00.000Z", "downloadUrl": "/files/abc123/document.pdf" } }

📋 List All Files

Get a list of all uploaded files (for development/testing)

GET /files

Request

curl -X GET http://localhost:3000/files

Response

{ "success": true, "files": [ { "id": "abc123", "filename": "document.pdf", "originalname": "document.pdf", "size": 1024000, "uploadDate": "2025-09-18T10:30:00.000Z", "expiryDate": "2025-09-19T10:30:00.000Z", "downloadUrl": "/files/abc123/document.pdf" } ] }

🗑️ Delete Files

Delete a specific file before it expires

DELETE /files/:id

Request

curl -X DELETE http://localhost:3000/files/abc123

Response

{ "success": true, "message": "File deleted successfully" }

⚠️ Error Handling

Common error responses and status codes

Status Code Error Description
400 Bad Request Invalid request format or missing files
404 Not Found File not found or expired
413 Payload Too Large File exceeds 100MB limit
500 Internal Server Error Server-side error occurred

Error Response Format

{ "success": false, "error": "File not found", "message": "The requested file does not exist or has expired" }
