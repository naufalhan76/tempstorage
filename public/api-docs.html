API Documentation

Mabox provides a simple REST API for temporary file storage with automatic expiration

🚀 Key Features

Latest improvements and automation-friendly features

  • 🔒 Secure Unique IDs: Files accessed via unpredictable 32-character hex IDs instead of filenames
  • 🤖 Automation-Friendly: Simple response mode returns plain text URL (not JSON) for n8n, Zapier, etc.
  • 📁 Original Filenames Preserved: Downloads use original filename while links remain secure
  • ⏰ Flexible TTL: Set custom expiration times from 30 minutes to 7 days
  • 📝 Custom Filenames: Override filename while preserving extension
  • 🧹 Auto-Cleanup: Expired files automatically removed from storage

📤 Upload Files

Upload one or multiple files to temporary storage

POST /upload

Request

Send files as multipart/form-data with optional parameters:

Parameters
Parameter Type Required Description
file File ✅ Yes The file to upload (max 200MB)
ttl String ❌ No Expiration time: 30m, 1h, 3h, 6h, 24h, 7d
Default: 30m
fileName String ❌ No Custom filename (will preserve original extension if not provided)
simple String ❌ No Set to "true" for plain text response. Returns URL string instead of JSON (perfect for automation).
Examples

Basic Upload:

curl -X POST https://mabox.tech/upload \ -F "file=@document.pdf"

Upload with TTL:

curl -X POST https://mabox.tech/upload \ -F "file=@document.pdf" \ -F "ttl=24h"

Upload with Custom Filename:

curl -X POST https://mabox.tech/upload \ -F "file=@document.pdf" \ -F "fileName=my-document"

Upload with All Parameters:

curl -X POST https://mabox.tech/upload \ -F "file=@document.pdf" \ -F "ttl=7d" \ -F "fileName=important-document.pdf"

Simple Response (For Automation Tools like n8n):

curl -X POST https://mabox.tech/upload \ -F "file=@document.pdf" \ -F "simple=true"

Standard Response

{ "success": true, "message": "File uploaded successfully", "data": { "fileName": "document.pdf", "originalName": "document.pdf", "fileSize": 1024000, "downloadUrl": "https://mabox.tech/files/a5b3c8e9f1d2a4b6c7e8f9a0b1c2d3e4", "expiresAt": "2025-09-19T10:30:00.000Z", "ttl": "30m" } }

Simple Response (when simple=true)

Content-Type: text/plain

https://mabox.tech/files/a5b3c8e9f1d2a4b6c7e8f9a0b1c2d3e4

Note: Returns plain text URL string, perfect for automation tools that need direct access to the download link.

📥 Download Files

Download files using their unique secure ID

GET /files/:uniqueId

Security Note: Files are accessed via unique, unpredictable IDs (32-character hex strings) rather than original filenames. This provides better security while preserving original filenames when downloaded.

Request

curl -X GET https://mabox.tech/files/a5b3c8e9f1d2a4b6c7e8f9a0b1c2d3e4 \ -o downloaded_document.pdf

Response

Returns the file content with appropriate headers:

  • Content-Type: Detected MIME type
  • Content-Disposition: attachment; filename="original-filename.pdf"
  • Content-Length: File size in bytes

ℹ️ Get File Information

Retrieve metadata about a specific file

GET /info/:uniqueId

Request

curl -X GET https://mabox.tech/info/a5b3c8e9f1d2a4b6c7e8f9a0b1c2d3e4

Response

{ "success": true, "data": { "originalName": "document.pdf", "size": 1024000, "uploadedAt": "2025-09-18T10:30:00.000Z", "expiresAt": "2025-09-19T10:30:00.000Z", "mimetype": "application/pdf" } }

⚠️ Error Handling

Common error responses and status codes

Status Code Error Description
400 Bad Request Invalid request format or missing files
404 Not Found File not found or expired
413 Payload Too Large File exceeds 200MB limit
500 Internal Server Error Server-side error occurred

Error Response Format

{ "success": false, "error": "File not found", "message": "The requested file does not exist or has expired" }
